# AI_TASK_LOG.md

## 현재 구조 요약

- 클라이언트는 로컬 카메라 0번을 열고 객체 탐지를 수행합니다.
- 서버는 프레임 탐지 결과와 `rule_config`를 기준으로 이벤트를 판정합니다.
- 서버는 이벤트 종료 시 프리뷰 버퍼에서 MP4 클립과 JPG 썸네일을 생성합니다.
- 뷰어는 서버 프리뷰 스트림, 이벤트 로그, 클립, 룰 설정을 다룹니다.

## 최근 반영된 흐름

- 원격 PC 클라이언트 구분을 위해 `source_key`에 클라이언트 소유자 정보를 포함합니다.
- 뷰어는 좌측 카메라 리스트에서 카메라를 선택/해제합니다.
- 선택된 카메라가 없으면 전체 이벤트, 선택된 카메라가 있으면 해당 이벤트만 표시합니다.
- 이벤트 로그에는 썸네일을 표시합니다.
- 이벤트 클릭 시 해당 카메라 타일에서 클립을 재생합니다.
- 위험구역 이벤트에는 당시 `danger_zone_roi`를 저장해 클립 재생 시 ROI를 오버레이합니다.
- ROI 드래그는 즉시 저장하지 않고 `위험구역 편집 종료` 시 저장합니다.
- 테스트용 `DB Clear`는 이벤트 DB/프레임 탐지 DB/클립/썸네일을 정리합니다.

## 운영 메모

- 빌드/실행 배치파일은 `subst`/junction을 쓰지 않습니다.
- 저장소는 C 또는 D 드라이브 루트 근처의 짧은 경로에 둡니다.
- 현재 기본 카메라 표시명은 뷰어 표시 순서 기준 `카메라 1`, `카메라 2`입니다.
- 장기적으로는 물리 카메라 슬롯과 현재 연결 클라이언트를 분리해야 합니다.
