# AI_RULES.md

## 기본 원칙

- 서버와 GUI 책임을 섞지 않는다.
- 새 기능은 먼저 서버 API 기준으로 설계한다.
- GUI는 가능하면 서버 조회와 로컬 재생만 담당하게 유지한다.
- 분석 코어 수정 시 기존 `VideoPipeline` 흐름을 함부로 깨지 않는다.

## 서버 수정 규칙

- 소스 등록/삭제/시작/중지는 `sources` 테이블과 worker manager를 함께 본다.
- 이벤트/프레임 탐지/상태 저장 포맷은 DB와 GUI 조회를 같이 고려한다.
- 클립은 파일 저장, 메타데이터는 DB/이벤트 payload로 관리한다.
- `source_key`와 `source_slug` 규칙은 서버와 GUI가 같이 기대하므로 확인 없이 바꾸지 않는다.

## 분석 코어 수정 규칙

- `safety_monitor_server/app/analysis/`는 서버 내부 분석 패키지로 간주한다.
- `NoHelmetRule`이 기대하는 클래스명(`person`, `helmet`, `head`) 호환성을 유지한다.
- 모델 교체 시 정확도뿐 아니라 GUI 오버레이, 이벤트 지속시간, 클립 생성 영향도 같이 본다.
- 성능 개선은 먼저 추론 FPS, 입력 크기, 비동기 저장 경로를 점검한다.

## GUI 수정 규칙

- 로컬 브리지 파일을 새로 되살리지 않는다.
- 이벤트 로그 필터는 “선택된 소스만 / 선택 해제 시 전체” 규칙을 유지한다.
- 로그 클릭, 클립 열기, 원본 복귀는 source_key 기준으로 일관되게 맞춘다.
- 멀티소스에서 비활성 플레이어 pause 규칙을 유지한다.

## 실행/검증 규칙

- 무거운 빌드나 장시간 분석 실행은 사용자가 직접 수행한다.
- 구조 변경 후에는 최소한 import/문법 수준 정합성은 먼저 확인한다.
- 서버 실행 경로는 `safety_monitor_server` 폴더 기준을 우선 가정한다.
