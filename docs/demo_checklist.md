# Demo Checklist

## 1. 사전 확인

- [ ] 저장소가 C/D 루트 근처의 짧은 경로에 있는지 확인
- [ ] `check_environment.bat` 실행 후 Python/Flutter/Visual Studio C++/Windows SDK 확인
- [ ] `install_dependencies.bat all` 성공 확인
- [ ] 클라이언트 모델 파일 확인: `safety_monitor_client/embedded_backend/app/analysis/models/weights/best.pt` 또는 `best.engine`
- [ ] 서버 PC IPv4 주소 확인
- [ ] 다른 PC에서 `http://<서버IP>:8000/health` 접근 가능 여부 확인

## 2. 서버

- [ ] `run_server.bat` 실행
- [ ] 서버 콘솔에 `0.0.0.0:8000` 바인딩 확인
- [ ] `GET /health` 응답 확인
- [ ] `GET /api/sources` 응답 확인
- [ ] 필요 시 `setup_server_firewall.bat` 관리자 권한 실행

## 3. 뷰어

- [ ] `run_viewer.bat` 실행
- [ ] 서버 URL 입력/적용 확인
- [ ] 좌상단 `DB Clear` 버튼 표시 확인
- [ ] `DB Clear` 실행 시 이벤트 DB/클립/썸네일 초기화 확인
- [ ] 좌측 카메라 리스트 표시 확인
- [ ] 카메라 리스트 항목 클릭 시 해당 영상 선택 확인
- [ ] 이미 선택된 카메라 다시 클릭 시 선택 해제 확인
- [ ] 선택된 카메라가 없으면 전체 이벤트 로그 표시 확인
- [ ] 선택된 카메라가 있으면 해당 카메라 이벤트만 표시 확인
- [ ] 이벤트 로그가 자체 스크롤 영역 안에서만 스크롤되는지 확인

## 4. 클라이언트

- [ ] `run_client.bat` 실행
- [ ] 원격 서버 URL에 서버 PC IPv4 주소 입력
- [ ] 서버 health check 성공 확인
- [ ] 카메라 0번 open 성공 확인
- [ ] 모델 로딩/TensorRT engine 준비 로그 확인
- [ ] 서버 뷰어에서 해당 카메라가 표시되는지 확인
- [ ] 다른 PC 클라이언트 추가 실행 시 카메라가 별도 항목으로 표시되는지 확인

## 5. 실시간 모니터링

- [ ] Live Monitoring에 영상이 표시되는지 확인
- [ ] 연결 상태 점이 초록/주황/빨강으로 표시되는지 확인
- [ ] 분석 시작 지연 중 진행 바 오버레이가 표시되는지 확인
- [ ] 정상 프레임/탐지 수신 후 진행 오버레이가 사라지는지 확인
- [ ] 영상 우상단 최대화 버튼 동작 확인
- [ ] 다시 누르면 원래 그리드로 복귀하는지 확인

## 6. 이벤트/클립

- [ ] 안전모 미착용 이벤트가 서버 DB에 저장되는지 확인
- [ ] 위험구역 이벤트가 서버 DB에 저장되는지 확인
- [ ] 이벤트 클립 MP4가 `safety_monitor_server/data/clips/`에 생성되는지 확인
- [ ] 이벤트 썸네일 JPG가 `safety_monitor_server/data/event_thumbnails/`에 생성되는지 확인
- [ ] 이벤트 로그에 썸네일이 표시되는지 확인
- [ ] 이벤트 클릭 시 해당 카메라 영상 타일에서 클립 재생 확인
- [ ] 클립 재생 중 우상단 닫기 버튼 표시 확인
- [ ] 닫기 버튼 클릭 시 클립 닫힘 확인
- [ ] 위험구역 이벤트 클립 재생 시 ROI 사각형 오버레이 확인

## 7. 룰 설정

- [ ] 룰 설정 패널이 이벤트 로그 아래에 있는지 확인
- [ ] 안전모 미착용 룰 ON/OFF 저장 확인
- [ ] 위험구역 룰 ON/OFF 저장 확인
- [ ] 위험구역 룰 OFF 상태에서도 ROI 드래그 편집 가능 확인
- [ ] ROI 드래그 후 바로 저장되지 않고 임시 표시만 되는지 확인
- [ ] `위험구역 편집 종료` 클릭 시 ROI 저장 확인
- [ ] ROI 저장만으로 위험구역 룰 토글이 자동 ON 되지 않는지 확인
- [ ] 다시 편집 진입 시 기존 저장 ROI가 표시되는지 확인

## 8. 종료/재실행

- [ ] 클라이언트 종료 후 뷰어 상태가 오프라인/숨김 정책대로 갱신되는지 확인
- [ ] 클라이언트 재실행 시 같은 카메라가 정상 재등록되는지 확인
- [ ] 서버 재실행 후 뷰어/클라이언트 재연결 확인
