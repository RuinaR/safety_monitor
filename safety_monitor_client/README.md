# Safety Monitor Client

클라이언트는 실제 카메라 소스를 다루고 객체 탐지를 수행하는 운영 앱입니다.

## 현재 운영 정책

- PC 1대당 클라이언트 1개 실행
- 클라이언트 1개당 로컬 `0`번 카메라 1개 사용
- 클라이언트는 서버에 자기 카메라를 하나의 소스로 등록
- 같은 PC에서 재실행해도 가능한 한 같은 `client_id`를 유지

## 클라이언트가 담당하는 일

- 로컬 `0`번 카메라 열기
- YOLO/TensorRT 기반 객체 탐지 실행
- 서버로 소스 정보 전송
- 서버로 상태 heartbeat 전송
- 서버로 프리뷰 프레임 전송
- 서버로 프레임 탐지 결과 전송
- 로컬 내장 백엔드 실행/관리

## 클라이언트가 소유하는 것

- 원본 카메라 접근 권한
- 로컬 YOLO 가중치
- TensorRT engine
- GPU 추론 실행
- 내장 분석 백엔드

## 클라이언트가 하지 않는 일

- 중앙 이벤트 DB 저장
- 위험 이벤트 최종 판정
- 이벤트 클립/썸네일 최종 저장
- 여러 클라이언트 전체 이벤트 통합 관리
- 뷰어용 중앙 스트림 허브 역할

위 작업은 서버가 담당합니다.

## 룰 설정

소스별 룰 설정은 서버 DB에 저장되며, 현재 UI 흐름에서는 뷰어에서 변경합니다.

- 안전모 미착용 룰 ON/OFF
- 위험구역 룰 ON/OFF
- 위험구역 ROI

룰 변경은 변경 시점 이후 새 이벤트 판정에 적용됩니다. 과거 이벤트 DB/클립은 그대로 조회 가능합니다.

## 실행

루트에서 실행합니다.

```bat
run_client.bat
```

직접 개발 실행할 때:

```bat
cd safety_monitor_client
..\flutter\bin\flutter.bat run -d windows
```
