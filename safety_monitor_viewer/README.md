# Safety Monitor Viewer

뷰어는 서버에 저장된 데이터를 보는 관제 앱입니다.

## 뷰어가 담당하는 일

- 서버 프리뷰 스트림 표시
- 좌측 카메라 리스트 표시/선택/선택 해제
- 소스별 연결 상태 점 표시
- 서버 이벤트 로그 표시
- 이벤트 썸네일 표시
- 서버 이벤트 클립 재생
- 위험구역 이벤트 클립 재생 시 이벤트 당시 ROI 오버레이
- 여러 클라이언트가 등록한 카메라를 한 화면에서 모니터링
- 서버에 저장되는 소스별 룰 설정 변경
- 테스트용 이벤트 DB/클립/썸네일 초기화

## 뷰어가 하지 않는 일

- 객체 탐지 실행
- 로컬 카메라 직접 열기
- 클라이언트 내장 백엔드 실행
- 로컬 가중치 관리
- TensorRT 엔진 관리

## UI 흐름

- 좌측 `카메라` 패널에서 카메라를 클릭하면 선택됩니다.
- 선택된 카메라를 다시 클릭하면 선택이 해제됩니다.
- 영상 타일 클릭도 같은 선택/해제 흐름을 사용합니다.
- 선택된 카메라가 없으면 이벤트 로그는 전체 이벤트를 보여줍니다.
- 선택된 카메라가 있으면 이벤트 로그는 해당 카메라 이벤트만 보여줍니다.
- 이벤트를 클릭하면 해당 카메라 타일에서 클립이 재생됩니다.
- 클립 재생 중 타일 우상단 닫기 버튼으로 클립을 닫습니다.

## 룰 설정

룰 설정 패널은 이벤트 로그 아래에 있습니다.

- 안전모 미착용 룰 ON/OFF
- 위험구역 룰 ON/OFF
- 위험구역 ROI 드래그 편집
- 위험구역 ROI 초기화

ROI 편집 정책:

- `위험구역 드래그 편집`을 누르면 편집 모드로 들어갑니다.
- 기존 ROI가 있으면 영상 위에 표시됩니다.
- 드래그 종료 시 서버에 바로 저장하지 않습니다.
- `위험구역 편집 종료`를 눌렀을 때 현재 ROI가 저장됩니다.
- ROI 저장만으로 위험구역 룰이 자동 ON 되지 않습니다.

## 정리

- 영상 원본 소유자: 클라이언트
- 이벤트 판정/저장 주체: 서버
- 관제/룰 설정 주체: 뷰어

## 실행

루트에서 실행합니다.

```bat
run_viewer.bat
```

개발 모드로 직접 실행할 때:

```bat
cd safety_monitor_viewer
..\flutter\bin\flutter.bat run -d windows
```
